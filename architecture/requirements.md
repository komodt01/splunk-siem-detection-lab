# Requirements

## Purpose

The purpose of this project is to design, deploy, and evaluate a Security Information and Event Management (SIEM) platform that demonstrates enterprise security monitoring concepts using Splunk Enterprise. The project emphasizes architectural decision making, operational visibility, and security detection rather than product-specific administration.

---

# Functional Requirements

The solution shall:

- Collect Windows security events.
- Collect Linux security events.
- Support centralized log aggregation.
- Support searching using SPL.
- Support dashboard creation.
- Support custom detection rules.
- Support incident investigation.
- Support simulated attack scenarios.
- Support documentation of architecture decisions.

---

# Non-Functional Requirements

The solution should:

- Minimize implementation cost.
- Be deployable on local hardware.
- Be reproducible.
- Be easy to maintain.
- Be modular.
- Support future expansion.
- Support additional log sources.

---

# Security Requirements

The solution shall:

- Follow least privilege where practical.
- Protect administrative access.
- Secure log transmission.
- Maintain auditability.
- Support investigation of security events.

---

# Business Requirements

The project should demonstrate:

- Security architecture thinking.
- SIEM fundamentals.
- Detection engineering.
- Incident investigation.
- Security monitoring concepts.
- Documentation suitable for interview discussions.

---

# Success Criteria

The project will be considered successful when it can:

- Collect Windows and Linux logs.
- Detect simulated attacks.
- Display meaningful dashboards.
- Demonstrate SPL searches.
- Investigate incidents.
- Explain architectural tradeoffs.
